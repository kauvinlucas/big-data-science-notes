Hive cheatsheet

List databases: show database;

Database structure: desc database <name>;

List tables: show tables;

Show table structure: desc <table name>; #Table constants and types

desc formatted <table name>; #See partitions and other info

desc extended <table name>; #Same as desc formatted, but with summarized info

Create database: create database <name>;

create database <name> location <path>; #Store on a different path than default (/user/hive/warehouse/)

create database <name> comment <description>; #Add comment (recommended)

Create table: create table <name> (<column 1> <data type 1>, <column 2> <data type 2>, …); #Add internal table (columns + metadata)

create external table <name> (<column 1> <data type 1>, <column 2> <data type 2>, …) location <path>;

Drop internal/external table: drop table; # when dropping external table, only metadata is deleted. Table is still in hdfs.

Examples of simple data types: INT, SMALLINT, TINYINT, BIGINT, FLOAT, DOUBLE, STRING, VARCHAR, CHAR.

Examples of complex data types: ARRAY (element list), MAP (key/value), STRUCT (more than one data type in a single item), UNION (not currently supported by Impala and Cloudera).

Define delimiters (csv files, optional): row format delimited fields terminated by ‘<delimiter>’;

row format delimited lines terminated by ‘<delimiter>’;

Skip first rows: tblproperties(“skip.header.line.count”=”<number of line>”);

Define location for external tables: location <directory>;

Example of table creation:

create external table user(

id int,

name String,

age int

)

row format delimited

fields terminated by ‘\t’

lines terminated by ‘\n’

stored as textfile

location ‘/user/cloudera/data/client’;

Insert data: insert into table <name> partition(<partition name>=’<value>’) values (<field>, <value>, …);

insert into <name> select <columns> from <table> where <condition>;

Insert data from directory: hive load data inpath ‘<directory>’ into table <table name>;

hive load data inpath ‘<directory>’ overwrite into table <table name> partition(<partition name>);

Select statement arguments: select * from <table>

<where …>

<group by...>

<having...>

<order by …>

<limit n>;

Joins: ANSI joins (inner join, left outer right outer, full outer)

Create view: create view <view name> as select <column> from <table name>;
